# UC-TASK-001: View and manage tasks

| | |
|---|---|
| **Actor** | User |
| **Preconditions** | Signed in |
| **Milestone** | M3 |
| **External request** | None |
| **LLM** | No |

## Context

Tasks are a flat collection. They can be created manually or generated by bundle
generation as Gap Tasks linked to a specific role and gap. All writes are direct
Firestore operations — no backend call needed.

## Flow

1. User opens `/tasks`.
2. SPA subscribes to `users/{uid}/tasks` via `onSnapshot`.
3. Tasks are grouped: open, in-progress, complete. Optional filter by linked role.
4. **Create**: user types description, optional due date, optional role link. SPA
   appends to Firestore.
5. **Complete**: user marks task done. SPA sets `completedAt`; `actualDays` is derived
   from `createdAt` diff in whole days.
6. **Delete**: confirmation prompt. Gap tasks linked to an active role show an
   additional warning ("This gap task is linked to an active role — deleting it will
   not reopen the gap.").

## Gap tasks

Gap tasks are created by bundle generation (via the "Create gap tasks" ThinkButton on
the role detail page). They arrive pre-linked with `source: gap`, `roleId`, and
`gapIndex`. Completing a gap task makes it eligible for reassessment
([UC-TASK-002](UC-TASK-002-reassess-gap-task.md)).

## Postconditions

- Task state reflects user actions.
- Completed gap tasks with `actualDays` populated contribute to Task Calibration once
  ≥ 3 such tasks exist.
