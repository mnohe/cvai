= CVAI Layout Packs
:toc:
:toclevels: 2

CVAI renders PDFs from Typst layout packs stored under `CVAI_DATA/pdf/layouts`.
Layouts are data assets, so private installations can
install or replace them without modifying the application container.

== Directory Layout

A layout pack is a directory with this minimum structure:

[source,text]
----
layout-pack/
  layout.yaml
  cv.typ
  fonts/          # optional
----

`layout.yaml` is the manifest:

[source,yaml]
----
id: compact
name: Compact
version: 1
entrypoint: cv.typ
fonts:
  - family: Archivo
    path: fonts/archivo
----

Rules:

* `id` must use lowercase letters, numbers, `_`, or `-`.
* `entrypoint` must name a file in the layout root, not a nested path.
* Declared font paths must exist inside the layout pack.
* Fonts should live inside the layout directory so the pack is portable.

== Importing

Import a local layout pack:

[source,sh]
----
cvai layouts import /path/to/layout-pack /path/to/private-data
----

Use `--replace` to overwrite an installed layout with the same ID:

[source,sh]
----
cvai layouts import --replace /path/to/layout-pack /path/to/private-data
----

The command copies the pack into:

[source,text]
----
CVAI_DATA/pdf/layouts/<layout_id>/
----

== Updating

For the current release, updates are performed by re-running the import command
with `--replace`.

The web layout manager will add source-aware updates:

* GitHub or GitLab layouts update with `git pull`.
* Regular URL layouts update by downloading the URL again.
* ZIP-uploaded layouts have no update action unless they are uploaded again.
* ZIP uploads that would overwrite an existing layout require confirmation.
